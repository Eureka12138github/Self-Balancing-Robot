## 功能概述

本系统支持通过平衡车自带的 **OLED 显示屏 + 4 个按键** 直接调整 PID 控制参数，无需连接电脑或手机。调整后的参数会自动保存到 Flash 存储器中，断电后不会丢失。

### 核心特性

✅ **板载调试** - 无需串口线或蓝牙模块  
✅ **实时显示** - OLED 屏幕实时显示当前参数  
✅ **掉电保存** - 参数保存到 Flash，重启不丢失  
✅ **精细调节** - 支持 0.01 精度微调  
✅ **安全保护** - 参数范围限制，防止误操作

## 硬件要求

| 硬件 | 状态 | 说明 |
|------|------|------|
| **STM32F103C8T6 最小系统板** | ✅ 必需 | 主控制器 |
| **OLED 显示屏（I2C 接口）** | ✅ 必需 | 128x64 分辨率 |
| **4 个独立按键** | ✅ 必需 | K1/K2/K3/K4 |
| **MPU6050 传感器** | ✅ 必需 | 姿态解算 |
| **电机驱动模块** | ✅ 必需 | PWM 输出 |

## 快速入门

### 操作流程概览

```
开机 → 主界面 → 按 K3(下) → "设置" → 按 K1(确认)
                                    ↓
                    选择 "Angle PID" → 按 K1 进入
                                    ↓
                      选择 "A_Kp" → 按 K1 进入编辑
                                    ↓
                        按 K2/K3 调整数值
                                    ↓
                      长按 K1(3 秒) 保存并退出
```

### 按键功能速查

| 按键 | 短按 | 长按 (>3 秒) |
|------|------|------------|
| **K1** | 确认/进入编辑 | 保存参数并退出 |
| **K2** | 减小数值 / 向上导航 | - |
| **K3** | 增大数值 / 向下导航 | - |
| **K4** | 返回上级 | - |

## 详细操作步骤

### 步骤 1：进入参数设置界面

1. **开机启动**
   - 打开电源开关
   - 等待 OLED 显示主界面

2. **导航到设置菜单**
   ```
   主界面显示:
   ├─ 设置
   ├─ 监控
   └─ 更多
   
   按 K3(下键) 选中 "设置"
   按 K1(确认) 进入
   ```

3. **选择 PID 类型**
   ```
   设置页面显示:
   ├─ RUN/STOP
   ├─ Turn PID    (转向环)
   ├─ Speed PID   (速度环)
   └─ Angle PID   (直立环) ← 推荐从此开始
   
   按 K3 选中 "Angle PID"
   按 K1 进入
   ```

---

### 步骤 2：调整 PID 参数

#### 2.1 进入编辑模式

```
Angle PID 页面显示:
├─ A_Kp:+12.345   ← 比例系数
├─ A_Ki:+0.150    ← 积分系数
├─ A_Kd:+15.600   ← 微分系数
├─ A_tar:+0.000   ← 目标角度
├─ A_act:+0.050   ← 实际角度
└─ A_out:+10.200  ← PID 输出

按 K2/K3 移动光标到 "A_Kp"
按 K1 短按 → 进入编辑模式（数值开始闪烁）
```

#### 2.2 调整数值

**编辑模式显示**（数值闪烁）：
```
├─ A_Kp:▶12.345◀  ← 左右箭头表示正在编辑
```

**调整方法**：
- **短按 K2**：减小 0.100（如 12.345 → 12.245）
- **短按 K3**：增大 0.100（如 12.345 → 12.445）
- **长按 K2/K3**：触发加速模式（快速连续变化）

**加速倍率表**：
| 按住时间 | 加速等级 | 实际步长 | 说明 |
|---------|---------|---------|------|
| 0-100ms | 1x | 0.100 | 慢速精细调节 |
| 100-200ms | 5x | 0.500 | 中速调节 |
| 200-300ms | 10x | 1.000 | 快速调节 |
| >300ms | 15x | 1.500 | 极速调节 |

#### 2.3 切换到其他参数

```
调整完 Kp 后：
- 按 K4(返回) → 退出编辑模式，回到参数列表
- 或继续按 K3 → 移动到下一个参数 "A_Ki"
```

**推荐调整顺序**：
1. 先调 **Kp**（比例）→ 建立基本站立能力
2. 再调 **Kd**（微分）→ 抑制振荡
3. 最后调 **Ki**（积分）→ 消除静差（通常设为 0）

---

### 步骤 3：保存参数到 Flash

⚠️ **重要**：调整后必须手动保存，否则重启后会丢失！

#### 方法 1：通过菜单项保存（推荐）

1. **退出编辑模式**
   - 按 K4 返回参数列表

2. **导航到"保存参数"**
   ```
   设置页面显示:
   ├─ RUN/STOP
   ├─ Turn PID
   ├─ Speed PID
   ├─ Angle PID
   └─ 保存参数 ← 新增菜单项
   
   按 K3 选中 "保存参数"
   按 K1 确认保存
   ```

3. **确认保存成功**
   ```
   OLED 显示 "[✓Saved]" 提示
   LED 闪烁 3 次（亮 100ms，灭 100ms）
   ```

#### 方法 2：快捷保存（当前未实现，可选开发）

长按 K1(>3 秒) 直接保存当前所有参数。

---

### 步骤 4：验证保存结果

1. **重启系统**
   - 关闭电源
   - 等待 5 秒
   - 重新上电

2. **检查参数**
   ```
   进入 "设置" → "Angle PID"
   查看 A_Kp/A_Ki/A_Kd 是否为之前保存的值
   ```

## 参数说明

### 直立环 PID（Angle PID）

控制小车前后倾斜的平衡，是最关键的参数组。

| 参数 | 符号 | 范围 | 默认步长 | 物理意义 | 典型值 |
|------|------|------|---------|---------|-------|
| **比例** | Kp | -20~20 | 0.100 | 对倾斜角度的响应强度 | 10~15 |
| **积分** | Ki | -2~2 | 0.010 | 消除长期静差 | 0~0.1 |
| **微分** | Kd | -20~50 | 0.100 | 抑制振荡，增加阻尼 | 10~20 |

**调参口诀**：
> 先调 Kp 能站稳，再调 Kd 不振荡，最后 Ki 消误差。

---

### 速度环 PID（Speed PID）

控制小车的前进/后退速度，影响行驶平顺性。

| 参数 | 符号 | 范围 | 默认步长 | 物理意义 | 典型值 |
|------|------|------|---------|---------|-------|
| **比例** | Kp | -20~20 | 0.100 | 速度误差响应 | 2~5 |
| **积分** | Ki | -2~2 | 0.010 | 消除稳态误差 | 0.05~0.2 |
| **微分** | Kd | -20~50 | 0.100 | 速度变化阻尼 | 0.5~2 |

---

### 转向环 PID（Turn PID）

控制小车的左右转向，影响直线行驶和转弯性能。

| 参数 | 符号 | 范围 | 默认步长 | 物理意义 | 典型值 |
|------|------|------|---------|---------|-------|
| **比例** | Kp | -20~20 | 0.100 | 方向偏差响应 | 3~8 |
| **积分** | Ki | -2~2 | 0.010 | 消除偏航 drift | 0~0.1 |
| **微分** | Kd | -20~50 | 0.100 | 转向阻尼 | 1~5 |

## 调试技巧

### 场景 1：小车无法站立（总是倒地）

**症状**：开机后小车无法保持直立，立即倒下。

**排查步骤**：
1. 检查 MPU6050 是否正常工作（进入"监控"页面查看角度数据）
2. 将 `A_Kp` 从默认值增加 20%（如 12.0 → 14.4）
3. 如果仍然倒地，继续增加 `A_Kp` 直到能短暂站立
4. 出现小幅振荡后，增加 `A_Kd` 抑制振荡

**经验值**：
```
初始设置:
A_Kp = 10.0
A_Ki = 0.0
A_Kd = 10.0

调整策略:
- 每次增加 A_Kp by 0.5
- 出现振荡后，每次增加 A_Kd by 1.0
```

---

### 场景 2：小车持续振荡（左右摇摆）

**症状**：小车能站立，但持续左右摇摆，振幅约 5-10 度。

**解决方案**：
1. **减小 Kp**：降低 10-20%（如 15.0 → 12.0）
2. **增大 Kd**：增加 20-30%（如 10.0 → 13.0）
3. **检查机械结构**：确认车轮安装牢固，无松动

**判断标准**：
- ✅ 正常：轻微抖动（<2 度），快速恢复
- ❌ 异常：大幅振荡（>5 度），持续不止

---

### 场景 3：小车跑偏（直线行驶时偏向一侧）

**症状**：小车在平地行驶时逐渐偏向左侧或右侧。

**解决方案**：
1. 进入 "Turn PID" 页面
2. 调整 `T_Kp`：
   - 向左偏：增加 T_Kp（如 5.0 → 6.0）
   - 向右偏：减小 T_Kp（如 5.0 → 4.0）
3. 如果效果不明显，调整 `T_Ki`（每次±0.05）

**测试方法**：
```
1. 在平坦地面标记起点
2. 让小车直线行驶 3 米
3. 测量横向偏差距离
4. 根据偏差方向调整参数
```

---

### 场景 4：参数调乱后恢复出厂设置

**情况**：参数调整不当，小车完全无法工作。

**恢复方法**：

#### 方法 1：手动恢复（当前可用）
1. 记下默认参数值（拍照或记录）
2. 重新上电（会加载 Flash 中的旧参数）
3. 逐个参数改回默认值
4. 保存

#### 方法 2：代码扩展（可选开发）
添加"恢复出厂设置"菜单项，一键清空 Flash 参数。

**默认参考值**：
```c
// 直立环
A_Kp = 12.500
A_Ki = 0.000
A_Kd = 15.000

// 速度环
S_Kp = 3.500
S_Ki = 0.100
S_Kd = 1.000

// 转向环
T_Kp = 5.000
T_Ki = 0.000
T_Kd = 2.000
```

## 常见问题

### Q1: 为什么调整后参数没有保存？

**A**: 可能原因：
1. ❌ 未按保存流程操作（只调整了数值，未调用保存）
2. ❌ 保存后立即断电（Flash 写入需要时间）
3. ❌ Flash 损坏（罕见，需检查硬件）

**解决方法**：
- 确保看到 "[✓Saved]" 提示后再断电
- 重启后检查参数是否保持

---

### Q2: 参数调整范围是多少？

**A**: 各参数范围如下：

| 参数 | 最小值 | 最大值 | 说明 |
|------|-------|-------|------|
| Kp (Angle) | -20.000 | +20.000 | 负值用于调试 |
| Ki (Angle) | -2.000 | +2.000 | 通常为正 |
| Kd (Angle) | -20.000 | +50.000 | 可大于 Kp |
| Kp (Speed/Turn) | -20.000 | +20.000 | 同上 |

超出范围的值会被自动限制在边界。

---

### Q3: 步长可以自定义吗？

**A**: 当前固定为以下配置：

| 参数类型 | 粗调步长 | 精细步长 |
|---------|---------|---------|
| Kp | 0.100 | - |
| Kd | 0.100 | - |
| Ki | - | 0.010 |

如需修改，编辑 `menu_data.c` 中的 `s_PID_Kp_edit_config` 等结构体。

---

### Q4: Flash 寿命是多久？

**A**: STM32F103 的 Flash 规格：
- **擦写次数**: ≥10 万次
- **数据保持**: ≥20 年（@25°C）

**建议**：
- 避免频繁保存（单次调试保存 1-2 次即可）
- 找到满意参数后再统一保存

---

### Q5: 可以在小车运行时调整参数吗？

**A**: ⚠️ **不建议**，原因：
1. 调整参数可能导致小车失控倒地
2. 分散注意力，存在安全隐患

**正确做法**：
1. 停车 → 调整参数 → 保存 → 重新测试
2. 如需实时观察，使用"监控"页面查看数据

## 技术细节

### Flash 存储布局

```
Flash 地址：0x0800FC00 (最后一页，1KB)

Store_Data[0]      = 0xA5A5          // 初始化标志
Store_Data[1-9]    = 系统参数        // 阈值、计数器等
Store_Data[10-36]  = 错误日志        // 3 条记录
Store_Data[37-39]  = 警报日志        // 预留
Store_Data[67-84]  = PID 参数        // 9 个参数×2 u16/参数
Store_Data[85-511] = 未使用          // 预留扩展
```

### PID 参数存储格式

每个 float 参数拆分为 2 个 uint16_t 存储（IEEE754 单精度）：

```c
float Kp = 12.345;

// 存储（小端模式）
Store_Data[67] = (uint16_t)(Kp的二进制低 16 位);
Store_Data[68] = (uint16_t)(Kp的二进制高 16 位);

// 读取
Kp = *(float*)&Store_Data[67];
```

### 数据验证机制

加载时进行三重验证：

1. **NaN 检查**：`(kp != kp)` 检测非数字
2. **范围检查**：`abs(kp) < 1000.0` 防止异常值
3. **校验和**：未来可扩展 CRC32 验证
